1. What is Semantic HTML? 
    2. What is HTML used for? 
        HTML is used to structure a web page. 
    3. What is an attribute and where do we put it?
         An attribute is a stylistic property of an element that is placed inside a tag. 
    4. What is the h1 tag used for? How many times should I use it on a page?
        An h1 tag is used to identitly the most important heading on a webpage and should be used once. 
    5. Name two tags that have required attributes
        <img> & <a>
    6. What do we put in the head of our HTML document? 
        The head section holds metadata about the webpage such as text encoding and links to CSS stylesheets or javascript files.
    7. What is an id?
        An id names an element and must be unique. It can then be used to apply unique styling to that element.
    8. What elements can I add an id to? 
        You can add an id to any element.
    9. How many times can I use the same id on a page? 
        Once. 
    10. What is a class? 
        A class can be applied to instances of the same element, such as a paragraph. 
    11. What elements can I add a class to? 
        Most, if not all elements.
    12. How many times can I use the same class on a page? 
        As many times as you need. 
    13. How do I get my link to open in a new tab?
        target="_blank"
    14. What is the alt attribute in the image tag used for? 
        For screen readers or displaying text when the image cannot be populated. 
    15. How do I reference an id?
        #id_name
    16. What is the difference between a section and a div
        a section is semantic or conveys some thematic meaning, a div does not. 
    17. What is CSS used for? 
        CSS is used to style your HTML page. 
    18. How to we select an element? Example - every h2 on the page
        h2 {}
    19. What is the difference between a class and an id? - Give me an example of when I might use each one
        A class can be given to multiple elements and an id can only be given to one. You could use a class on h2 elements so they are all styled the same. You may want to use an id one of these elements, if it needs to be styled in a seperate manner. 
    20. How do we select classes in CSS?
        .class_name{}
    21. How do we select a p element with a single class of “human””?
        p.human{}
    22. What is a parent child selector? When would this be useful? 
        A parent child selector is a way to select a tag that is nested inside another tag. This would be usefule when you are attempting to style an element that exists outside of a nested context. 
    23. How do you select all links within a div with the class of sidebar?
        .sidebar.div.a{}
    24. What is a pseudo selector?
        A selector that is used to style elements in a certain state, such as, hover. 
    25. What do we use the change the spacing between lines?
        line-height: 
    26. What do we use to change the spacing between letters?
        letter-spacing:
    27. What do we use to to change everything to CAPITALS? lowercase? Capitalize?
        text-transform: text-transform: uppercase;
        text-transform: lowercase;
        text-transform: capitalize;
    28. How do I add a 1px border around my div that is dotted and black?
        div{
            border-width: 1px;
            border-style: dotted;
            border-color: black;
        }
    29. How do I select everything on the page?
        *{}
    30. How do I write a comment in CSS?
        /* Your comment here */
    31. How do I find out what file I am in, when I am using the command line? 
        pwd
    32. Using the command line - how do I see a list of files/folders in my current folder?
        ls
    33. How do I remove a file via the command line? Why do I have to be careful with this? 
        rm, you have to be careful because you can't undo this. 
    34. Why should I use version control? 
        version control allows you to keep chronological snapshots of your work. It is also much easier to work with a team of developers in this manner as multiple people will be working on their own branch before merging back to remote. 
    35. How often should I commit to github?
        Every 15 minutes or so.

    37. Walk me through Lambda's git flow.
        1. Fork
        2. Clone and head into your local folder
        3. Checkout your own branch
        4. Work on your repo
        5. Push new changes to your branch (git add . -> git commit -m "your helpful message here"
